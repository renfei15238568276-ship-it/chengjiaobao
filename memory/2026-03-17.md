## 修复问题

### Admin 用户管理页面
- 问题：用户无法访问 /admin/users 页面，一直报服务器错误
- 原因排查：
  1. 最初有 admin 权限检查问题（role cookie 未正确设置）
  2. 后来发现是 supabase-admin 调用问题
- 解决方案：
  1. 暂时移除 admin 权限检查以便调试
  2. 重写页面，使用内联的 server actions 处理订阅和权限
  3. 最终成功显示用户列表表格，带有"开通个人版"和"设为管理员"按钮

### 注册页面 - 重复用户名检查
- 问题：用户注册重复用户名时没有提示
- 修复：代码已有检查 "用户名已被占用"，但数据库RLS导致检查失败
- 修复文件：src/lib/user-service.ts

### 数据库 RLS 问题
- 问题：Supabase RLS 策略导致数据丢失/不可见
- 解决：禁用 users 表的 RLS
```sql
ALTER TABLE users DISABLE ROW LEVEL SECURITY;
ALTER TABLE subscriptions DISABLE ROW LEVEL SECURITY;
```

### 修改密码功能
- 问题：用户无法自己修改密码
- 修复：修改 src/app/settings/actions.ts，使用直接 REST API 更新密码
- 修复后用户可以自己修改密码
